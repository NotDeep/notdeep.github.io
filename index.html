以后的机器学习向博文都想以What,How,Application,Optimization的思路

What

K-means是基于数据划分的一种无监督聚类算法

聚类算法可以理解为无监督的分类方法

无监督的分类方法就是样本集预先不知道所属类别或者标签,需要根据样本之间的距离or相似程度进行自动的分类
基于的办法有基于划分的方法、基于联通性的方法、基于概率分布模型的方法等。
K-means属于基于划分的聚类方法

其实也蛮好理解,我们可以想想一下放在平面坐标系中的一些点,如何把这些点划分到各种类别。我们可以人为的制定一定的标准,比如满足某个限制就属于某个类别,就好比我们考90分叫优秀,70~90叫良,60~70叫及格一样。而K-means是一种自动分类的方法,他是基于数据间的“距离”和相似度来进行划分的,这里的距离在不同的场景或许可以代表具体的含义。

How

假设样本集合$D = { \{x_j\}_{j=1}^n }$,${x_j\in{R^d}}$,K-means聚类算法想达到的就是将数据集划分为$k(k<n)$类:$S={\{S_1,S_2,…,S_k\}}$,使划分后的k个子集和满足类内的误差平方和最小:
$$l_{k-means}(S) = arg_{S={ \{S_i\} }^k_{i=1}} min\sum_{i=1}^{k}\sum_{x\in S_i} {|x-c_{i}|^2_2}$$

其中:
$$c_i=\frac{1}{ |S_i| }\sum_{x\in S_i}x$$

Latax初学者,打公式打得好辛苦

求解目标函数$l_{k-means}$是一个NP-hard(惊!)问题,无法保证得到一个绝对的全局最优解(why,复杂度爆炸啦)。在常用和经典的求解方法中都是采取一种迭代优化的策略。

一般步骤

  1. 随机选取K个种子点,K个簇(如何选取K是玄学,后面会讨论一下下)
  2. 然后对所有数组求到这个K个种子点的距离,假如$X_i$离种子点$S_i$最近,那么$X_i$就属于这个$S_i$这个簇
  3. 根据步骤2的分配结果,更新聚类中心:
    $${c_i^{(t+1)} = \frac{1}{ |S_i| }\sum_{x_j \in {S_i^{t}}}} x_j$$
    迭代第t+1轮的各个簇等于第t轮的各个簇的点求个平均
  4. 若迭代次数完成或者前后两次迭代的差小于设定的阈值$\epsilon$,即
    $${|c_i^{(t+1)}-c_i^{(t)}|^2_2<\epsilon}$$

则算法结束,否则重复执行步骤2、3

求点群中心的算法

上面就很简单的算各个点的X\Y坐标的平均值,另外还有三个求中心点的公式:

  1. Minkowski Distance公式——λ可以随意取值，可以是负数，也可以是正数，或是无穷大
    $$d_{ij} = {\sqrt[\lambda]{\sum_{k=1}^n {|x_{ik}-x_{kj}|^{\lambda}} }}$$
  2. Euclidean Distance公式——也就是第一个公式λ=2的情况
    $$d_{ij} = {\sqrt{\sum_{k=1}^n {|x_{ik}-x_{kj}|^{2}} }}$$
  3. CityBlock Distance公式——也就是第一个公式λ=1的情况
    $$d_{ij} = \sum_{k=1}^{n}|x_{ik}-x_{kj}|$$

这里的$\lambda$的取值就就是反应一种怎么趋近于这个簇的中心的

Application

K-means是比较简单的一个算法。
但是我们可以把问题考虑到更复杂的应用场景,比如对于人群的分类,我们就可以考虑人的多个属性了,年龄、性别、爱好、学历…而具体的问题目测也需要不一样的距离计算函数。
我们只要把现实中的一些问题抽象为若干维的描述属性,我们就可以思考是否能够能用K-means做个分类。
现实中我们借此来分析商品的相似度,归类商品就是一个很好的运用
