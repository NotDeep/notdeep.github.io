Problem A. Tricky Sum

题意

对$1$~$n$求和,但是$2$的$n$次幂都变成减。

分析

$n$范围有$1e9$,直接$for$肯定$GG$,直接等差数列求和为$sum$,然后剔除掉$2$的$n$次幂,$sum$到答案每一个$2$的$n$次幂都要被剔除两倍才对。

Problem B. Queries on a String

题意

给一个字符串,然后给出若干个$l_i,r_i,k_i$表示把串的$l_i$到$r_i$这个子串循环右移$k_i$次,问最后串变成什么样

分析

经典做法,假设前$n-k$个字符为$A$,后$k$个字符为$B$,假设逆置操作为$’$,字符串进行循环右移$k$位等价于先将$A$进行逆置操作$A’$,再将$B$进行逆置操作$B’$,最后对于整体再做一次逆置操作即$(A’B’)’=BA$。这样就完成了循环右移$k$次操作。
STL里面有个$ratate$函数可以直接完成一段交换的操作,效率不是很好,这里够了。。

Problem C. Nearest vectors

题意

给一堆以原点发射出来的向量,按顺序标号,问相距最近的两个向量的标号。

分析

本来是一个水题,直接对极角排序就好了。但是这里卡$double$精度,需要$long$ $double$。几何的精度是深坑！taotao又给我普及了下$atan$和$atan2$函数的区别,据说三角函数库基本都是泰勒公式求的,精度不是很好。$atan2$用的神奇的姿势实现具有更好的精度,有空可以研究一下。
